Quadratics Roots Calculator

Quadratic Roots Calculator

Personal Coding Project with JS

This project may be viewed on my Github

GitHub

PROBLEM STATEMENT:
After the Garage internship I realized the importance of being able to speak the same technical language with the developers, as it greatly increases the teams' efficiency. Thus, I started learning Javascript over the summer, and this is my first coding project with it.

While brainstorming for an idea on what is a simple enough project to start with, I remembered my first and second year taking my math prerequisites for my physics courses. Many quadratic calculations had to be made, but there was a lack of good mobile apps to check my calculations with. The only ones available are poorly made apps that involves typing in a form. The poor user flow of these apps were frustrating and really slows me down in completing my calculations. Furthermore, these apps return inexact, decimal solutions and cannot compute imaginary solutions, which is useless to me in a lot of the quadratic solutions' applications. Thus, I have decided to make a calculator that satisfies these needs.

DECREASE IN USER ACTIONS TO INCREASE EFFICIENCY:
Existing apps are unecessarily tedious: it usually involves with inputting a form on mobile. Thus the goal of my design exercise is to lessen the number of steps to create a more seamless UX and increase the user's efficiency. This is considered as the use case scenario is of a user making large amounts of calculations per sitting.

CODE
The entire calculation is handcoded in Javascript. The outcome is divided into 3 possibilities according to quadratic relations:

  • 2 Real Solutions
  • Repeated Roots
  • No Real Solution

The quadratic formula has 2 parts, a fraction and a square root. each of these sections are calculated in seperate functions and simplified to its lowest common denominator using recursions. Depending on how many solutions it has, "if" and "else" statements are used to display the solution in the proper mathematical form.
